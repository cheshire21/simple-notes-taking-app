# Product Roadmap

## Phase 1: Authentication

- User registration (email + password)
- User login with JWT tokens
- Logout with token blacklisting
- Protected routes — redirect unauthenticated users to login

## Phase 2: Dashboard Layout

- Main dashboard layout: categories sidebar (left) + notes list (right)
- Responsive layout following Figma design
- Top-right "Add Note" button

## Phase 3: Categories

- List all categories for the authenticated user (sidebar)
- Create a new category
- Delete a category
- Filter notes list by selected category

## Phase 4: Notes

- List all notes (or filtered by category)
- Create a note via full-page modal (title, content, category)
- Edit a note via full-page modal
- Delete a note

## Phase 5: Polish

- Loading states and error handling throughout
- Empty states (no notes, no categories)
- Form validation feedback
- Design accuracy pass against Figma
